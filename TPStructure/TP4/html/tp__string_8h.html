char* read_one_line ( FILE *  file)

Lit une ligne dans un fichier texte.

Paramètres
[in,out]fileUn flux vers le fichier à lire
Précondition
Le flux file est ouvert.
Renvoie
Soit NULL, soit une chaîne de caractères str
Postcondition
Le flux file est toujours ouvert.
Si la fonction renvoie NULL, alors le flux file était déjà à la fin du fichier avant l'appel (rien à lire).
Sinon, str contient tous les caractères depuis la position courante jusqu'au premier saut de ligne suivant (s'il existe) exclus, ou jusqu'à la fin du fichier (sinon).
Dans ce cas, le flux file avance d'autant de caractères (placé soit après le saut de ligne, soit à la fin du fichier).
Avertissement
Si la fonction ne renvoie pas NULL, les ressources mémoire occupées par str doivent être libérées en appelant free().
Remarques
Cette fonction optimise l'espace mémoire utilisé par str (le cas échéant).

int qsort_compare_strings ( const void *  lhs,
const void *  rhs 
)

Compare deux chaînes de caractères (critère de tri)

Paramètres
[in]lhsUne chaîne de caractères (left-hand-side ; passée par adresse via un pointeur générique)
[in]rhsUne chaîne de caractères (right-hand-side ; passée par adresse via un pointeur générique)
Renvoie
Un entier result dont le signe permet de comparer lhs et rhs :
  • Si result est négatif, alors lhs précède rhs dans l'ordre lexicographique.
  • Si result est nul, alors lhs et rhs sont égales.
  • Si result est positif, alors lhs suit rhs dans l'ordre lexicographique.
Note
Cette fonction peut être utilisée comme critère de tri par la fonction qsort().
Remarques
Cette fonction se contente d'appeler strcmp() en typant et déréférençant lhs et rhs de manière adéquate.

)

Trie et compte des lignes dans un fichier.

Paramètres
[in]in_filenameLe nom du fichier à analyser (à lire)
[out]out_filenameLe nom du fichier dans lequel écrire les résultats
Précondition
in_filename désigne le nom d'un fichier texte existant.
L'extension de in_filename et out_filename est : .txt
Note
Si le fichier out_filename n'existe pas, il est créé, sinon son contenu est effacé.
Renvoie
Le nombre de lignes lues dans le ficher in_filename
Postcondition
out_filename contient les lignes (deux à deux distinctes) du fichier in_filename, triées dans l'ordre lexicographique, précédées du nombre d'occurrences dans in_filename.
Voir également
qsort_compare_strings() (critère de tri des chaînes de caractères lues)

)

Copie une chaîne de caractères, en remplaçant la première occurrence d'une sous-chaîne par une autre sous-chaîne.

Paramètres
[out]destLa chaîne de caractères résultante
[in]srcLa chaîne de caractères initiale
[in]beforeLa sous-chaîne à rechercher dans src
[in]afterLa sous-chaîne à substituer à before
Avertissement
Le comportement de cette fonction est indéfini si dest ne peut pas contenir suffisamment de caractères !
Valeurs retournées
Vrai(valeur non nulle) Si before apparaît au moins une fois dans src
Faux(valeur nulle) Sinon
Postcondition
dest est une copie de src dans laquelle la première occurrence de before (si elle existe) est remplacée par after.

size_t read_input_array ( char ***  array)

Enregistre un nombre quelconque de chaînes de caratères tapées par l'utilisateur.

Paramètres
[out]arrayL'adresse d'un tableau dynamique de chaînes de caratères dynamiques
Avertissement
Le pointeur à l'adresse array doit être initialisé à NULL avant l'appel !
Renvoie
Le nombre de chaînes de caractères (lignes) tapées par l'utilisateur
Note
La saisie s'arrête lorsque l'utilisateur entre la chaîne de caractères (ligne) vide.
Postcondition
Le tableau à l'adresse array contient les chaînes de caractères tapées par l'utilisateur (en console).
La fin du tableau à l'adresse array est représentée par la valeur NULL dans la dernière case.
Avertissement
Les ressources mémoire occupées par le tableau à l'adresse array doivent être libérées en appelant free_input_array().
